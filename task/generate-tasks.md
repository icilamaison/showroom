# Rule: PRD로부터 작업 목록(Task List) 생성하기

## 목표
완성된 PRD 마크다운 파일을 입력으로 받아, **주니어 개발자가 한 단계씩 따라 구현할 수 있는** 상세한 단계별 작업 목록(Task List)을 생성하도록 가이드한다.

이 규칙은 3-step 워크플로우의 **Step 2**다. (Step 1: `create-prd.md`, Step 3: `process-task-list.md`)

---

## 입력

사용자가 참고할 PRD 파일을 지정한다. (예: "`/tasks/prd-contract-system.md` 보고 작업 목록 만들어줘")

PRD가 없는 상태에서 이 규칙을 적용하지 않는다. PRD 없이 작업 목록을 만들면 누락되거나 PRD 의도와 다른 작업이 생길 위험이 크다.

---

## 진행 과정 (2단계, 사용자 확인 필수)

이 과정은 **반드시 2단계로 나눠서** 진행한다. 한 번에 상위 작업과 하위 작업을 동시에 쏟아내지 않는다.

### 1단계: 상위 작업(Parent Tasks) 생성
1. PRD를 분석한다.
2. 구현에 필요한 **상위 작업을 약 5개 내외**로 생성한다. (이 시점에서는 하위 작업을 만들지 않는다.)
3. 상위 작업만 사용자에게 제시한다.
4. 다음과 같이 안내한다: *"상위 작업 목록을 생성했습니다. 하위 작업까지 만들어드릴까요? 진행하시려면 'Go'라고 답해주세요."*
5. **사용자가 'Go'(또는 동의하는 답)를 보내기 전까지 2단계로 넘어가지 않는다.**

### 2단계: 하위 작업(Sub-Tasks) 생성
1. 사용자 확인을 받은 후, 각 상위 작업을 더 작은 단위의 구체적인 하위 작업으로 분해한다.
2. 각 하위 작업은 **한 번에 구현 가능한 크기**여야 한다 (한 세션 안에 끝낼 수 있는 수준).
3. 하위 작업은 PRD의 디테일(데이터 모델, API 명세, Acceptance Criteria)을 반영하고, 논리적인 구현 순서를 따른다.

### 3단계: 관련 파일(Relevant Files) 식별
- 구현 과정에서 생성되거나 수정될 것으로 예상되는 파일 목록을 작성한다.
- 코드 파일마다 대응하는 테스트 파일도 함께 표시한다.
- 파일별로 한 줄 설명을 붙인다.

### 4단계: 파일로 저장
- 결과물은 `/tasks/tasks-[prd-파일명].md` 형식으로 저장한다.
- 예: PRD가 `prd-contract-system.md`라면 → `tasks-prd-contract-system.md`

---

## 출력 형식

```markdown
## Relevant Files

- `path/to/file1.ts` - 설명 (어떤 역할을 하는 파일인지)
- `path/to/file1.test.ts` - file1.ts에 대한 단위 테스트
- `path/to/file2.tsx` - 설명

### Notes
- 단위 테스트는 가능하면 테스트 대상 코드 파일과 같은 디렉토리에 둔다.
- 테스트 실행 명령어 등 참고사항을 적는다. (예: `npm test`)

## Tasks

- [ ] 1.0 상위 작업 제목
  - [ ] 1.1 하위 작업
  - [ ] 1.2 하위 작업
  - [ ] 1.3 하위 작업
- [ ] 2.0 상위 작업 제목
  - [ ] 2.1 하위 작업
  - [ ] 2.2 하위 작업
```

---

## 대상 독자 및 작성 원칙

- 작업 목록은 주니어 개발자가 PRD를 다시 읽지 않아도, 이 목록만 보고 구현을 진행할 수 있을 정도로 구체적으로 작성한다.
- 각 하위 작업은 "무엇을 어디에" 수준까지 구체적으로 적는다. ("API 구현" 같은 모호한 표현 대신 "POST /api/contracts 엔드포인트에 서버 측 validation 추가" 식으로.)
- 하위 작업 순서는 실제 구현 순서(예: DB 스키마 → API → 프론트엔드)를 따른다.

---

## 최종 지침

1. **1단계(상위 작업)까지 만든 뒤 반드시 멈추고 사용자의 'Go' 확인을 기다린다.** 확인 없이 하위 작업까지 한 번에 만들지 않는다.
2. 작업 목록이 완성되면 곧바로 구현을 시작하지 않는다. Step 3(`process-task-list.md`)의 규칙에 따라 한 작업씩 진행한다.
